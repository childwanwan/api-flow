- 请求超时: 30000ms
- 最大重试次数: 64次

## 7. 技术实现难点提示

### 7.1 插件热部署技术难点

| 难点 | 描述 | 建议方案 |
|------|------|---------|
| 类加载隔离 | 插件类不能污染系统类加载器 | 使用独立的 ClassLoader 加载插件 |
| 版本切换事务 | 插件切换时正在执行的任务怎么办 | 正在执行的任务继续使用旧版本插件 |
| 故障隔离 | 一个插件崩溃不能影响其他插件 | 使用独立线程池执行插件逻辑 |
| 资源释放 | 插件卸载时需要清理占用的资源 | 提供插件的 `destroy` 生命周期钩子 |

### 7.2 配置热更新注意事项

| 注意事项 | 说明 |
|---------|------|
| 原子性 | 配置更新时保证一致性，不能出现部分更新 |
| 回滚能力 | 更新失败时能快速回滚到上一版本 |
| 正在执行的任务 | 已经提交的任务继续使用提交时的配置快照 |

### 7.3 补偿机制实现要点

| 要点 | 说明 |
|------|------|
| 补偿数据存储 | 各插件保存的补偿数据需要序列化保存 |
| 倒序执行 | 补偿时需要按执行步骤的倒序进行 |
| 超时控制 | 补偿也需要有超时控制，避免无限等待 |
